Racket Parser & Evaluator

This project was a part of Formal Languages and Functional Programming (CS301). The goal was to create a parser and an evaluator that could work in tandem to evaluate expressions in the procedure first format utilized by Racket. First, I had to create a parser that could parse strings using a context-free-grammar. From this point I created an evaluator that could take the formatted equations from parser, and get the answer. The evaluator has the capacity of handling simple operations such as addition, subtraction, multiplication and division as well as more complex operations such as conditionals, mapping, and even lambdas. To see the entirety of my code and details of the project, check out my Github repository where I've posted everything, including a test file with example inputs and expected outputs.

Deadwood

In this project, I worked with a partner to implement a boardgame called Deadwood in java. First as a text user interface, and then as a graphic user interface. We handled the GUI portion with the use of javaFX. The project required us to practice many important and common design patterns such as singleton, adapter, and observer design patterns. Deadwood significantly strengthed my coding skills while enforing crucial object oriented design principles, including SOLID principles. As part of this project I had to create an XML Parser to turn data from XML format into game objects, which was another skill I was happy to pick up. Overall the project was interesting and enjoyable, especially due to my interest in board games.

Pente

This project is a work in progress. Pente is a two player abstract strategy game, with many similarities to the popular game Go. In Pente players take turn placing stones onto a grid, attempting to either get five in a row, or capture five pairs of their opponents stones. For this project I first implemented Pente in java as a text-user-interface, then as a graphic-user-interface using javascript/react. Pente is a fairly straightforwards game and I used it as an early project to help build my base java skills, as well as get a start to projects I can showcase. I plan to create a functional adaption of my pente project using React in the future.
