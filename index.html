    Projects

    Game-Rating Site

    For this project, I explored uncharted territory in backend development, continuing my trend of progressively raising the complexity of my projects. The result was a dynamic web application with advanced capabilities, including user differentiation for guests and full-access users, dynamic data display drawn from a MongoDB database, and posting of additional data to the backend via forms. Employing React for the client-side interface, I taught myself and acquired expertise in Express, Node.js, and MongoDB for the server-side architecture. The integration of a MongoDB database allows for an ever growing collections of games, users, and ratings to be stored and utilized for a variety of site opereations. Notable operations include the addition of games to the game list or ratings for games within this list that are then immediately rendered and accessible. This project was a chance for me to both establish important backend skills and work further with front-end skills that I've been developing with previous projects.

    Weather-App

    To enhance my proficiency in web development and React, I focused on a project to design a weather data application. Initially, I leveraged the OpenWeatherMap API to access global weather data for cities around the world. Simultaneously, I accessed data from GeoNames , which supplied a comprehensive text file containing the names of cities worldwide. From here I parsed the file and implemented a filtering mechanism to extract pertinent data, constructing the list that users can access for weather information. Furthermore, I employed virtualization techniques and conditional rendering to enhance the application's loading speed, ensuring swift and seamless interactions. Finally, I deployed the application using Firebase, making it accessible to anyoneas a web-app here.

    Pente

    Pente is an abstract strategy game for two players, bearing many similarities to the popular game Go. In Pente, players take turns placing stones onto a grid, aiming to achieve either five stones in a row or capture five pairs of their opponents' stones for victory. I first implemented Pente in Java as a text-user-interface, and later recreated the entire game as a graphical-user-interface through React, elevating the game's presentation and accessibility. Moreover, I enhanced the game's playability by adding restart and undo functionalities. Most recently I utilized Firebase to host the app, allowing anyone to play it here.

    Deadwood

    In this project, my partner and I collaborated to implement a board game called Deadwood in Java. We began by creating a text-user-interface and later expanded it to include a graphical-user-interface using JavaFX. Throughout the project, we practiced essential design crucial object-oriented design principles, including SOLID principles, as well as patterns like singleton, adapter, and observer. I also developed an XML parser to convert XML data to game objects. To ensure a comprehensive approach, we created complete use-case diagrams, write-ups, sequence diagrams, and class responsibility diagrams.

    Racket Parser & Evaluator

    This project was a part of Formal Languages and Functional Programming (CSCI 301) involved creating a parser and evaluator that worked together to evaluate expressions in the procedure-first format utilized by Racket. Initially, I developed an evaluator capable of handling various operations, from basic addition, subtraction, multiplication, and division, to more complex ones like conditionals, mapping, and lambdas. Following the evaluator, I created a parser that could parse strings using a context-free grammar to format the equations. The parser provided the formatted equations to the evaluator which would then map the characters to functions and process them to reach an answer.
